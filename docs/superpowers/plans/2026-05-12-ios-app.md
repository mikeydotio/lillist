# Lillist Plan 8 — iOS / iPadOS App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the iOS / iPadOS Lillist app — a single SwiftUI universal target with adaptive layouts (tab bar on iPhone, sidebar/`NavigationSplitView` on iPad), a Share Extension, and an App Intents target — all consuming `LillistCore` (Plans 1–5) and `LillistUI` (Plan 7).

**Architecture:** A single Xcode project `Apps/Lillist-iOS.xcodeproj` generated by `xcodegen` (matching Plan 7's tooling choice), added to the workspace established in Plan 7. The app target uses the SwiftUI lifecycle (`@main App`). UI is implemented as iOS-specific shells that compose components from `LillistUI`: on compact size class the shell is a `TabView` with Today / All / Filters / Search; on regular size class the same screens are presented in a `NavigationSplitView` with a sidebar. Task detail is a `TabView(.page)` within the navigation stack. A floating "+" overlay drives Quick Capture as a sheet. The Share Extension and App Intents target both link `LillistCore` and reuse its `CLIBridge` so behavior is uniform across surfaces. The app group `group.io.mikeydotio.Lillist` is the only persistence boundary touched by the three targets.

**Tech Stack:** Swift 6, SwiftUI (iOS 18 / iPadOS 18), App Intents, UniformTypeIdentifiers, LinkPresentation, `xcodegen`, `swift-snapshot-testing` (for UI snapshot tests, as established by Plan 7). No new third-party dependencies beyond Plan 7's choices.

**Design references:** See `docs/plans/2026-05-12-lillist-design.md`, especially **Section 7 (UI Structure) — iOS subsection**, plus Section 7 "Quick capture surfaces", Section 7 "Cross-platform conventions / keyboard shortcuts", and Section 7 "Pre-installed defaults". Section 4 (Notifications) and Section 5 (Smart Filters & Rule Engine) describe behaviors this app surfaces.

## Notes for the Implementer

**Adaptive UI policy.** The single rule is `horizontalSizeClass == .regular` ⇒ `SplitShell`, otherwise `TabShell`. This matches design Section 7's "iPadOS adaptive UI" requirement and means iPad landscape and most iPad portrait orientations land on `SplitShell` automatically. Split View on iPad and Slide Over compress to `TabShell`. There is no manual toggle.

**Shared components vs iOS-only.** Cells (task rows, tag chips, status indicators), composers (notes editor), and predicate authoring views are in `LillistUI` (Plan 7 co-owned). Shells, sheets, and platform-specific overlays (floating "+", iOS sync badge) live in this app's sources. New shared components added by this plan live under `Packages/LillistUI/Sources/LillistUI/iOS/`.

**Bundle ID and product unity.** All three targets share product family `io.mikeydotio.Lillist`:
- App: `io.mikeydotio.Lillist`
- Share Extension: `io.mikeydotio.Lillist.Share`
- App Intents target: `io.mikeydotio.Lillist.Shortcuts`

The macOS app (Plan 7) uses the same parent `io.mikeydotio.Lillist`. App Store treats them as one product.

**App Group.** All three iOS targets must include `group.io.mikeydotio.Lillist` for shared Core Data store + Defaults. CloudKit container is `iCloud.com.mikeydotio.lillist` (matches design Section 3).

**Pre-installed defaults.** Smart filters from design Section 7 (Today, This Week, No Tags, Recently Closed, Stale) are CloudKit-synced (Plan 2). Bootstrap creates them only when no `SmartFilter` rows with those names exist. If the macOS app already installed them and sync has caught up, we no-op. We guard the bootstrap behind `AppDefaults.installIfNeeded()` invoked in `LillistApp` body's `.task`.

**Accessibility & Dynamic Type.** Every view we ship sets explicit `accessibilityLabel`/`accessibilityHint` on interactive elements. Status indicators get a label like "Status: started, tap to advance". Tag chips read as "Tag: Work". We never hard-code line counts or pixel sizes — favor `lineLimit(nil)` and `minimumScaleFactor` only where layout demands it. Dark mode comes "for free" by using semantic colors (`Color.primary`, `Color.secondary`, etc.); explicit `colorScheme` use is forbidden except in snapshot tests.

**Testing.** Snapshot tests for steady states only — interaction states are integration tests. We snapshot in both iPhone 16 and iPad 13" sizes, light + dark. Integration tests run against an in-memory `PersistenceController` (Plan 1's facility). App Intent tests directly invoke `perform()` against the same in-memory store.

**Commits.** Each task ends in a commit. Use conventional-commit prefixes (`feat:`, `test:`, `chore:`, `fix:`, `refactor:`).

**Verification command throughout:**
```
xcodebuild -workspace Lillist.xcworkspace -scheme Lillist-iOS \
  -destination 'platform=iOS Simulator,name=iPhone 16' test
xcodebuild -workspace Lillist.xcworkspace -scheme Lillist-iOS \
  -destination 'platform=iOS Simulator,name=iPad Pro 13-inch (M4)' test
```

```swift
import SwiftUI
import LillistCore

/// Small badge surfaced in the top-right of each primary view on iOS.
/// Mirrors design Section 8's sync-indicator semantics.
public struct SyncStatusBadge: View {
    public var status: SyncStatus

    public init(status: SyncStatus) {
        self.status = status
    }

    public var body: some View {
        Circle()
            .fill(color)
            .frame(width: 10, height: 10)
            .overlay(
                Group {
                    if case .inProgress = status {
                        ProgressView()
                            .scaleEffect(0.5)
                    }
                }
            )
            .accessibilityLabel(label)
    }

    private var color: Color {
        switch status {
        case .syncedRecently: return .green
        case .inProgress: return .clear
        case .idle: return .yellow
        case .error: return .red
        }
    }

    private var label: String {
        switch status {
        case .syncedRecently: return "Synced"
        case .inProgress: return "Syncing"
        case .idle: return "Idle"
        case .error(let msg): return "Sync error: \(msg)"
        }
    }
}
```

```swift
#if os(iOS)
import SwiftUI
import LillistCore

/// Single-line text field used by Quick Capture and inline create.
///
/// Recognizes `#tag` and `^date phrase` tokens, surfaces autocomplete chips
/// below the field. Submission emits a parsed `QuickCaptureDraft`.
public struct QuickCaptureField: View {
    @Binding var text: String
    var tagSuggestions: [String]
    var dateSuggestions: [String]
    var onSubmit: (QuickCaptureDraft) -> Void

    public init(
        text: Binding<String>,
        tagSuggestions: [String] = [],
        dateSuggestions: [String] = [],
        onSubmit: @escaping (QuickCaptureDraft) -> Void
    ) {
        self._text = text
        self.tagSuggestions = tagSuggestions
        self.dateSuggestions = dateSuggestions
        self.onSubmit = onSubmit
    }

    public var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            TextField("Capture a task…", text: $text, axis: .horizontal)
                .textFieldStyle(.roundedBorder)
                .submitLabel(.done)
                .accessibilityIdentifier("QuickCaptureField")
                .onSubmit {
                    let draft = QuickCaptureDraft.parse(text)
                    onSubmit(draft)
                }
            if !tagSuggestions.isEmpty || !dateSuggestions.isEmpty {
                ScrollView(.horizontal, showsIndicators: false) {
                    HStack(spacing: 6) {
                        ForEach(tagSuggestions, id: \.self) { tag in
                            Text("#\(tag)").padding(.horizontal, 8).padding(.vertical, 4)
                                .background(Capsule().fill(Color.accentColor.opacity(0.15)))
                                .onTapGesture { text += " #\(tag)" }
                        }
                        ForEach(dateSuggestions, id: \.self) { phrase in
                            Text("^\(phrase)").padding(.horizontal, 8).padding(.vertical, 4)
                                .background(Capsule().fill(Color.orange.opacity(0.15)))
                                .onTapGesture { text += " ^\(phrase)" }
                        }
                    }
                }
            }
        }
    }
}
#endif
```

```swift
/// Single source of truth for live stores and monitors during a session.

/// Constructed once at scene boot and stamped into the environment tree.

@MainActor
@Observable
public final class AppEnvironment {
    public let persistence: PersistenceController
    public let taskStore: TaskStore
    public let tagStore: TagStore
    public let journalStore: JournalStore
    public let attachmentStore: AttachmentStore
    public let preferencesStore: PreferencesStore
    public let filterStore: SmartFilterStore
    public let syncMonitor: SyncStatusMonitor
    public let notifications: NotificationScheduler

    public init(
        persistence: PersistenceController,
        syncMonitor: SyncStatusMonitor,
        notifications: NotificationScheduler
    ) {
        self.persistence = persistence
        self.taskStore = TaskStore(context: persistence.viewContext)
        self.tagStore = TagStore(context: persistence.viewContext)
        self.journalStore = JournalStore(context: persistence.viewContext)
```

```swift
/// Bootstraps the five built-in smart filters described in design
/// Section 7, "Pre-installed defaults (deletable)". Idempotent: if a filter
/// with the same name already exists (e.g. from macOS app sync), no-ops.
enum AppDefaults {
    static let defaultFilterNames: [String] = [
        "Today", "This Week", "No Tags", "Recently Closed", "Stale"
    ]

    @MainActor
    static func installIfNeeded(env: AppEnvironment) async {
        do {
            let existing = Set(try env.filterStore.allFilters().map(\.name))
            for spec in builtIns where !existing.contains(spec.name) {
                _ = try env.filterStore.createFilter(
                    name: spec.name,
                    predicateGroup: spec.predicates,
                    sortField: spec.sort,
                    sortAscending: spec.ascending,
```

```swift
import Foundation
import LillistCore

/// Wraps `LillistCore.QuickCaptureDraft.parse` for use by the iOS sheet
/// and Share Extension. Recognizes `#tag` and `^date phrase` tokens; the
/// remaining text becomes the title.
enum QuickCaptureParser {
    struct Parsed: Equatable {
        var title: String
        var tagNames: [String]
        var deadlinePhrase: String?
    }

    static func parse(_ input: String) -> Parsed {
        var titleTokens: [String] = []
        var tags: [String] = []
        var datePhrase: String?

        let tokens = input.split(separator: " ", omittingEmptySubsequences: true)
        for token in tokens {
            if token.hasPrefix("#") {
                let name = String(token.dropFirst()).lowercased()
                if !name.isEmpty { tags.append(name) }
            } else if token.hasPrefix("^") {
                let phrase = String(token.dropFirst())
                if !phrase.isEmpty { datePhrase = phrase }
            } else {
                titleTokens.append(String(token))
            }
        }
        return Parsed(
            title: titleTokens.joined(separator: " "),
            tagNames: tags,
            deadlinePhrase: datePhrase
        )
    }
}
```

```swift
import Foundation
import UniformTypeIdentifiers

#if canImport(UIKit)
import UIKit
#endif

/// Decodes the heterogeneous payload an extension context delivers into a
/// shape the share sheet UI can pre-fill.
struct SharePayload {
    enum Item {
        case text(String)
        case url(URL)
    }

    struct Decoded {
        var suggestedTitle: String
        var notes: String?
        var url: URL?
    }

    private let items: [Item]

    init(extensionContext: NSExtensionContext?) {
        var collected: [Item] = []
        for input in extensionContext?.inputItems as? [NSExtensionItem] ?? [] {
            for provider in input.attachments ?? [] {
                if provider.hasItemConformingToTypeIdentifier(UTType.url.identifier) {
                    if let url = try? provider.loadItem(forTypeIdentifier: UTType.url.identifier) as? URL {
                        collected.append(.url(url))
                    }
                } else if provider.hasItemConformingToTypeIdentifier(UTType.plainText.identifier) {
                    if let text = try? provider.loadItem(forTypeIdentifier: UTType.plainText.identifier) as? String {
                        collected.append(.text(text))
                    }
                }
            }
        }
        self.items = collected
    }

    static func makeStub(items: [Item]) -> SharePayload {
        var s = SharePayload(extensionContext: nil)
        s = SharePayload(items: items)
        return s
    }

    private init(items: [Item]) { self.items = items }

    func decode() async throws -> Decoded {
        var title = ""
        var notes: String?
        var url: URL?
        for item in items {
            switch item {
            case .text(let str):
                let trimmed = str.trimmingCharacters(in: .whitespacesAndNewlines)
                title = String(trimmed.prefix(80))
                if trimmed.count > 80 { notes = trimmed }
            case .url(let u):
                url = u
                if title.isEmpty { title = "Link: \(u.host ?? u.absoluteString)" }
            }
        }
        return Decoded(suggestedTitle: title, notes: notes, url: url)
    }
}
```
